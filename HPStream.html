HPStream

Algorithm created by Charu C. Aggarwal, Jiawei Han, Jianyong Wang and Philip S. Yu in 2004 ## Goal of the HPStream algorithm:

  • Find a (k + 1)-partition of the data (last partition are outliers)
  • Find a set of dimensions for each cluster s.t. the mean cardinality of the sets is equal to l (parameter).

Fading cluster structure

  • Half life of a point t0 of a point: f(t0)=(1/2)f(0) (rate of decay of the weight assigned to each data point).
  • Data point weight: f(t)=2−λt
  • Fading cluster structure of a set of points 𝒞 = {Xi1, …, Xin} with time stamps Ti1…Tin: FC(𝒞)=(FC2x(𝒞, t),FC1x(𝒞, t),W(t)):
    • FC2x(𝒞, t)=∑k = 1nf(t − Tik)(xikj)2
    • FC1x(𝒞, t)=∑k = 1nf(t − Tik)xikj
    • $W(t) = \sum_{k=1}^n f(t-T_{i_k})$
    • ℬ(𝒞): bit vector of relevant dimensions.
  • Properties of fading structure:
    • FC(𝒞1 ∪ 𝒞2, t)=FC(𝒞1, t)+FC(𝒞2, t)
    • If no points are added in the time interval (t, t + δt), then FC(𝒞, t + δt)=e−λδtFC(𝒞, t)

HPStream Algorithm

  • Initialization:
    • Receive InitNumber Points
    • Compute k clusters using K-means
    • Select only l dimensions per cluster using the same method
    • Recompute an assignment according to the subspaces
    • Repeate until convergence
  • Normalization:
    • Compute σd std along each dimension for an initial sample of the datapoints
    • Devide the coodrinate of each new point by the std.
    • Recompute σd at an interval of N′ points
    • If σd changes to σ′d update fading clusters:
      • FC2x(𝒞, t)←FC2x(𝒞, t)×σd2/σ′d2
      • FC1x(𝒞, t)←FC1x(𝒞, t)×σd/σ′d
  • Update the temporal decay function for each cluster:
    • Let t denote the current time and tup the last update time for a given cluster. Multiply each fading cluster by: e−λ(t − tup)

  • Compute l relevant dimensions(FCS, l, X)

    • Create |FCS| tentative fading clusters by ading X to each existing Fading cluster (to avoid selecting only the dimensions of very small clusters)
    • Compute the radius along each dimension
    • Pick the |FCS × l| dimensions with least radii
    • Create the bit vector ℬ(𝒞r) for each fading cluster
  • Compute projected (segmental manhattan) distances to each fading cluster and find the closest center.
  • Compute the limiting radius (mean of the STD of the cluster measured along its subspace only multiplied by τ).
  • If the new point is within the radius add it, otherwise create a new fading cluster.
  • In order to keep constant the number of clusters, remove the one to which the least recent updating was performed

  • Remove clusters with 0 dimensions assigned to them

Parameters

  • Decay rate (λ) : 0.5
  • spread radius factor (τ) : 2
  • InitNumber : 2000
  • Average number of projected dimensions (l) : 20 (intrusions), 8 (covertype)
  • speed : 200 points per time unit and horizon H = 1
  • speed : 100 points per time unit and horizon H = 10
  • Number of clusters: Was set equal to the number of micro-clusters in Clustream, which is equal to 10 times the real number of clusters.
  • Std update frequence (N’)
